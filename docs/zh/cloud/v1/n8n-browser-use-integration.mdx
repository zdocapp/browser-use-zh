> **TL;DR** – 只需 **3 分钟**，您即可拥有一个 n8n 工作流，该工作流可以：
>
> 1.  显示一个表单，询问竞争对手的名称
> 2.  启动一个 Browser Use 任务，该任务会爬取网页并提取**定价、招聘信息、新功能及公告**
> 3.  通过**Webhook**等待任务完成
> 4.  格式化输出并将一条富文本消息发送到 Slack

您可以在下方获取工作流 JSON – 复制它并将其导入 n8n，填入您的 API 密钥，然后点击 _执行_ 🚀。

---

## 为什么在 n8n 中使用 Browser Use？

• **自主浏览** – Browser Use 像真实用户一样打开页面、跟踪链接、点击按钮并读取 DOM 内容。

• **结构化输出** – 您可以_精确地_告诉代理您需要哪些字段。无需脆弱的正则表达式或 XPath。

• **轻松扩展** – 启动数百个任务并通过 Cloud API 监控它们。

n8n 将所有环节连接起来，让您的团队能够即时获取数据——无需 Python 脚本或 CRON 作业。

## 前置条件

1.  **Browser Use Cloud API 密钥** – 从您的[账单页面](https://cloud.browser-use.com/billing)获取一个。
2.  **n8n 实例** – 自托管或 n8n.cloud。（下面的截图使用的是 n8n 1.45+。）
3.  **Slack 入站 Webhook URL** – 在您的 Slack 工作区中创建一个。

将两个密钥添加到 n8n 的凭证管理器：

```env title=".env example"
BROWSER_USE_API_KEY="sk-…"
SLACK_WEBHOOK_URL="https://hooks.slack.com/services/…"
```

## 工作流运行原理

### 1. `表单触发器` – 收集竞争对手名称

包含单个必填字段的公开 n8n 表单。用户提交时，工作流立即触发

### 2. `HTTP 请求 – Browser Use 运行任务`

我们向 `/api/v1/run-task` 发送 POST 请求，包含以下请求体：

```json title="run-task payload"
{
  "task": "Do exhaustive research on {{ $json[\"Competitor Name\"] }} and extract all pricing information, job postings, new features and announcements",
  "save_browser_data": true,
  "structured_output_json": {
    "pricing": {
      "plans": ["string"],
      "prices": ["string"],
      "features": ["string"]
    },
    "jobs": {
      "titles": ["string"],
      "departments": ["string"],
      "locations": ["string"]
    },
    "new_features": { "titles": ["string"], "description": ["string"] },
    "announcements": { "titles": ["string"], "description": ["string"] }
  },
  "metadata": { "source": "n8n-competitor-demo" }
}
```

关键部分：

• `structured_output_json` 告知代理返回哪些键值 – 无需后处理
• 我们通过 `metadata.source` 标记任务，以便 webhook 仅筛选_我们的_任务

### 3. `Webhook` + `条件判断` – 等待任务完成

当任务状态发生变化时 Browser Use 会发送 webhook（设置详情请参阅[Webhooks 指南](/cloud/v1/webhooks)）。我们在 `/get-research-data` 暴露 n8n Webhook 节点，让代理调用它

仅当**同时**满足以下条件时才继续执行：

- `payload.status == "finished"`
- `payload.metadata.source == "n8n-competitor-demo"`

### 4. `获取任务详情`

webhook 主体包含 `session_id`。我们获取完整任务记录以获得包含第2步结构化 JSON 的 `output` 字段

### 5. `代码节点 – 生成 Slack 消息`

一段简短的 JS 代码片段将 JSON 转换为格式美观的 Slack 消息块，包含表情符号和项目符号。请随意调整格式。

### 6. `HTTP 请求 – 发送至 Slack`

最后我们将消息通过 POST 请求发送到您的入站 webhook，然后庆祝 🎉。

## 按需自定义

此工作流仅是一个起点——Browser Use + n8n 为您提供无限可能。以下是一些建议：

| 想要实现...                       | 如何操作                                                                                                                              |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| **提取不同数据**       | 编辑 `structured_output_json` 以精确指定所需字段（价格、评论、联系信息等），并调整 JS 格式化器。 |
| **发送到 Teams/邮箱/Notion**   | 将最后的 Slack 节点替换为 Teams、Gmail 或 n8n 的 400+ 连接器中的任意一个。                                                               |
| **自动运行**            | 将表单触发器替换为 Cron 触发器，以实现每日/每周的竞争对手监控。                                                      |
| **监控多个竞争对手** | 使用 Google Sheets 触发器配合公司列表，并循环遍历它们。                                                               |
| **添加 AI 分析**              | 将提取的数据通过 OpenAI/Claude 进行处理，以生成洞察和摘要。                                                         |
| **创建警报**                | 设置条件逻辑，仅在竞争对手宣布新功能或价格变动时发送通知。                                          |
| **构建仪表板**            | 将数据发送到 Airtable、Notion 或 Google Sheets，以构建实时竞争对手情报仪表板。                                   |

Browser Use 的妙处在于，它处理复杂的网页浏览工作，而您可以专注于构建符合需求的完美工作流程。
